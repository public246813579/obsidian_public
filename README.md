# XY精密搬送ステージ 設計ナレッジ デモ

フォルダ（tree）構造のドキュメント管理では、設計変更の波及を見落とす。
本デモは **「リニアガイドのベンダー変更」** という**一見軽微なコストダウン**が、機構・電装・熱・検査・品質と**異部門に連鎖**していく様子を、Obsidianの **Graph view** と **Backlinks** でのみ発見できる形で再現したもの。

## 使い方

1. 右上の「グラフビュー」アイコンをクリック → 全ノードの関係を俯瞰
2. [[ECR-101 リニアガイド ベンダー変更]] を開く → 右サイドバーの**バックリンク**を展開
3. 下の4つの気付きを辿る
4. 議事録 [[MTG-2026-03-09 週次定例]] から時系列で読むのもおすすめ

## Obsidianでのみ気付ける4つのケース

| # | 起点ノート | Obsidianで浮上すること | tree/議事録では… |
|---|-----------|--------------------|-----------------|
| 1 | [[ECR-101 リニアガイド ベンダー変更]] | 取付面高さ0.3mm差 → [[CALC-101 XY公差スタック]] → [[SPEC-101 ピック配置精度]] を割る | 部品・計算・仕様が別フォルダで結線されない |
| 2 | [[PRT-101 リニアガイド]] | ブロック幅+2mm → [[PRT-105 ケーブルベア]] → [[CALC-103 ケーブルベア屈曲半径]] → [[FMEA-102 ケーブル断線]] → [[SPEC-104 MCBF]] | 機構と電装が別部門／別フォルダで完全分断 |
| 3 | [[PRT-109 出荷検査治具]] | 旧ガイド基準で設計された治具が使えない ← バックリンクで ECR-101 がぶら下がる | 検査部のフォルダにしか無く設計者から見えない |
| 4 | [[CALC-102 熱変位解析]] | [[PRT-103 サーボモータ]] 位置オフセット → ベース膨張方向変化 → [[SPEC-103 稼働温度範囲]] 逸脱 | 機構×熱の異ドメイン干渉はハブ経由でしか到達不可 |

## 議事録を介した"見えない連鎖"

4本の週次定例議事録を読むと、各週の議題は別々の問題に見える。
しかし [[PRT-101 リニアガイド]] のバックリンクを開くと、**4週の議事録が一本の糸で繋がる** — これがtree/議事録アーカイブでは到達不可能な気付きの本丸。

- [[MTG-2026-03-09 週次定例]] ECR起票、「機構影響は軽微」発言
- [[MTG-2026-03-16 週次定例]] 承認（電装・検査部門欠席）
- [[MTG-2026-03-23 週次定例]] 試作NG発覚
- [[MTG-2026-04-06 週次定例]] 別案件として報告された2件が実は同根
